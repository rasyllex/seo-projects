# ТЗ: Скилл съёма позиций

## Бизнес-задача
Клиент хочет отслеживать позиции своего сайта по 1000+ ключевым словам ежедневно.

## Критерии приёмки
1. Скилл читает Excel с колонкой `keyword`
2. Для каждого ключа запрашивает XMLRiver API
3. Находит позицию указанного домена в выдаче
4. Кэширует результаты на 24 часа
5. Сохраняет результат в Excel с колонками: keyword, position, url, date
6. Выводит статистику: всего ключей, найдено в ТОП-10, средняя позиция
7. Валидирует входные данные перед обработкой
8. Работает в многопоточном режиме (ThreadPoolExecutor)
9. Использует ретраи при временных ошибках API

## Ограничения
- Регион по умолчанию: 213 (Москва)
- Если API недоступен — fallback на мок-данные
- Кэш хранится в `data/cache.json`
- Количество потоков по умолчанию: 8

## Edge-cases
- Домен не найден в выдаче → position = 0
- API вернул ошибку → логируем, переходим к следующему ключу
- Кэш просрочен → перезапрашиваем
- Входной Excel без колонки `keyword` → ошибка с пояснением
- Пустые строки в Excel → пропускаем
- Домен передан как полный URL (`https://www.example.com/`) → корректно нормализуем

## Архитектура
- `run.py` — оркестратор
- `fetcher.py` — запросы к API (многопоточные, с ретраями)
- `cache.py` — кэширование
- `validate_input.py` — валидация входных данных
