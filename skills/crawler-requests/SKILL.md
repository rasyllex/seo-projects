# Скилл: Краулер (requests)

## Назначение
Многопоточный обход сайта: HTML-кэш + реестр зон (title, h1, h2, description) в Excel. Опции: ротация User-Agent, прокси (один или список с ротацией).

## Архитектура
- `run.py` — CLI и запись Excel.
- `crawler.py` — обход «волнами»: фронтир раздаётся `ThreadPoolExecutor` (по умолчанию 8 потоков), у каждого потока своя `requests.Session` (`threading.local`), пауза `--delay` соблюдается в каждом потоке. Потокобезопасность по построению: `visited` пополняется до раздачи волны, результаты собирает главный поток.
- `parser.py` — BeautifulSoup: внутренние ссылки (без якорей/параметров) и зоны.
- `cache.py` — HTML-кэш `../../data/crawler-requests/cache/<домен>/<md5(url)>.html`; при повторном запуске страница берётся с диска.

## Ротация User-Agent (`--rotate-ua`)
Пул из ~10 реальных UA (Chrome/Firefox/Safari/Edge/YaBrowser, Win/Mac/Linux), случайный выбор на каждый запрос. По умолчанию выключено — используется один дефолтный UA.

## Прокси
Приоритет: `--proxy` → `--proxy-file` (ротация: случайный на запрос) → `PROXY` из `config.env`.
Формат: `http://user:pass@host:port` (requests поддерживает auth-прокси нативно).
Кейс: сайты РФ, блокирующие зарубежный трафик — российский VPS со Squid в `config.env`, и краулер ходит с российского IP. В логах прокси маскируется (`http://***:***@host:port`).

## Ретраи
`--retries` (по умолчанию 3): повтор на таймаут, сетевую ошибку и 5xx с нарастающей паузой (2с, 4с, 6с). 4xx считается окончательным — не ретраится. После исчерпания попыток страница пропускается с пометкой в логе.

## Статусы
`status_code` — реальный код ответа. Страницы с ошибками (4xx/5xx/сеть) логируются и в реестр не попадают.

## Зависимости
requests, beautifulsoup4, lxml, pandas, openpyxl, python-dotenv
